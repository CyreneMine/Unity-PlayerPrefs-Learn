# AGENTS.md

## 项目定位

本项目是 Unity 学习项目，用于跟随唐老狮教程学习 `PlayerPrefs` 以及后续教程中的独立练习工程。

项目目标不是快速完成成品，而是记录学习过程、理解 Unity API、保留练习代码、方便后续复盘。

## Codex 工作原则

除非我明确要求，否则你只能进行：

- 阅读代码
- 检查代码
- 分析问题
- 给出修改建议
- 解释 Unity / C# / API 原理
- 总结学习内容
- 编写 README / 学习记录 / 复盘文档

禁止在未经我明确允许的情况下直接修改任何代码文件。

## 代码修改限制

默认情况下，不允许直接修改：

- `.cs` 脚本
- Unity 场景文件
- Prefab 文件
- ProjectSettings
- Packages 配置
- 任何会影响项目运行结果的文件

如果发现问题，请先告诉我：

1. 问题在哪里
2. 为什么会有问题
3. 推荐怎么改
4. 修改后的参考代码片段

只有当我明确说出以下类似指令时，才可以直接改代码：

- “直接修改”
- “帮我改”
- “按你的方案改”
- “可以动代码”
- “直接修复”
- “提交修改”

## 教程练习题检查规则

我完成教程中的练习题后，你需要帮我检查：

- 功能是否符合题目要求
- 代码逻辑是否正确
- 是否有明显 bug
- 命名是否清晰
- 是否有重复代码
- 是否有更好的写法
- 是否符合 Unity 常见实践

检查时优先解释原因，不要只给结论。

## GitHub 管理规则

当我完成一个阶段学习后，你可以帮我：

- 整理 README
- 更新 LearningProgress.md
- 总结本次学习内容
- 生成 commit message
- 提醒我是否需要提交 Git

除非我明确要求，否则不要自动 commit，也不要自动 push。

当我明确要求 push 时，需要先总结本次提交内容，再执行 Git 操作。

## 学习记录要求

建议维护以下文档：

- `README.md`：项目说明
- `LearningProgress.md`：学习进度
- `Notes/`：知识点笔记
- `Screenshots/`：运行效果截图，可选

每次阶段总结可以包含：

- 学习日期
- 学习主题
- 涉及 API
- 完成的练习
- 遇到的问题
- 解决方案
- 我的理解总结

## 回答风格

回答时请尽量：

- 使用中文
- 解释清楚原理
- 结合 Unity 场景说明
- 不要直接跳到复杂架构
- 不要为了“高级”而过度封装
- 优先帮助我理解为什么这样写

如果我写错了，不要只说错，要告诉我错在哪里，以及正确思路是什么。

## 特别强调

本项目是学习项目。

你的主要职责是辅助我理解、检查、复盘和管理，而不是替我完成代码。

除非我明确要求，否则不要代写完整功能代码，不要直接改动项目代码。
